# Team Agreements & Conflict Resolution

This document serves as the central hub for our team's operational rules, communication standards, and conflict management protocols.

## 1. Conflict Resolution Matrix
When disagreements occur (e.g., architectural decisions, workload disputes, or unresponsiveness), follow this escalation path:

| Escalation Level | Action to Take | Expected Timeframe |
| :--- | :--- | :--- |
| **Level 1: Direct Discussion** | Address the issue directly with the team member via a scheduled Discord/Teams call (not just text). | Within 24 hours |
| **Level 2: Team Consensus** | Bring the issue to the next Daily Stand-up. Present facts and vote on a technical or operational direction. | Next Stand-up |
| **Level 3: Supervisor Intervention** | If consensus cannot be reached, or a member is consistently unresponsive, document the attempts and loop in the project supervisor. | After 48 hours of blockage |

## 2. Communication Standards
* **WhatsApp / Instant Messaging:** Used exclusively for urgent blockers, quick clarifications, and meeting coordination. **Do not use WhatsApp for official project decisions.**
* **GitHub Issues / PR Comments:** Used for ALL technical discussions, design decisions, and code reviews. If it's not on GitHub, it didn't happen.
* **Meeting Notes:** All meeting notes must be committed to the `documents/meetings/` repository folder within 24 hours of the meeting concluding.

## 3. Blocker Protocol
If you cannot proceed with a task:
1. Move the Issue card to the **Blocked** column on the GitHub Project Board immediately.
2. Add a comment to the GitHub Issue detailing **why** it is blocked and **who/what** you are waiting on.
3. This must be done within 24 hours of being blocked.

## 4. Agile Ceremonies
We conduct the following ceremonies to maintain alignment:

| Ceremony | Frequency | Expected Output |
| :--- | :--- | :--- |
| **Sprint Planning** | Start of each sprint | Sprint issues assigned; story points agreed; Sprint Milestone updated. |
| **Daily Stand-Up** | Every 2–3 days | Brief async update posted in team channel: (1) Done yesterday, (2) Doing today, (3) Blockers. |
| **Sprint Review** | End of each sprint | Demo of completed artefacts to supervisor; update project board. |
| **Retrospective** | End of each sprint | 3 things that went well, 3 to improve; logged in GitHub Wiki (or central doc). |
