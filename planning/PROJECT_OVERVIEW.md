# Project Overview (SENG 31242)

This document provides a cohesive, high-level summary of the SENG 31242 System Design Project requirements, timeline, and tooling expectations.

## 1. Project Nature
The course simulates a real-world software engineering environment. Generic or "toy" applications are strictly prohibited. The project must fall into one of two categories:
* **Option A – Real Client:** Solving a genuine software problem for a business, NGO, or individual acting as the product owner. *(BK Restaurant falls here)*
* **Option B – Open-Source Contribution:** A significant new feature, refactor, or extension for an established open-source project.

## 2. Required Tooling
All project management and version control must be conducted professionally using the following tools:
* **GitHub Organisation:** The central host for all code, documentation, and artefacts.
* **GitHub Projects (Kanban):** Used for sprint boards, backlog management, and milestone tracking.
* **GitHub Issues:** Used for task tickets, bugs, and feature requests.
* **GitHub Milestones:** Used to define sprint and phase boundaries.
* **GitHub Wiki:** Used for living project documentation (e.g., meeting notes, retrospectives).

## 3. Project Timeline (8 Weeks)

The project is structured into 4 sequential Sprints, culminating in a complete design package ready for the development phase (SENG 34213).

| Sprint / Week | Phase | Key Activities & Deliverables |
| :--- | :--- | :--- |
| **Sprint 1 (Weeks 1–2)** | Idea Discovery & Pitch | Form team, identify problem space, pitch to supervisor panel, establish GitHub Org and initial backlog. |
| **Sprint 2 (Weeks 3–4)** | Requirements Engineering | Conduct fact-finding sessions, write use cases, draw activity diagrams, and draft the Software Requirements Specification (SRS). |
| **Sprint 3 (Weeks 5–6)** | System Design | Finalize SRS. Define system architecture, class diagrams, sequence diagrams, wireframes, and draft the Software Design Specification (SDS). |
| **Sprint 4 (Weeks 7–8)** | Consolidation & Submission | Finalize SDS, update backlog, conduct final design presentation, and submit the Final Report. |

## 4. Assessment & Passing Criteria
To pass SENG 31242 and qualify for the Semester 2 development phase, students must achieve **at least 40%** in every individual assessment component. Failure in any single component triggers a mandatory re-submission capped at a 'C' grade.
